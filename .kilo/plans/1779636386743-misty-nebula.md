# Análisis de Seguridad - Proyecto Oficinas

## Hallazgos Críticos

### 1. EXPOSICIÓN DE SECRETOS EN .env
**Severidad: CRÍTICA**
- El archivo `.env` contiene credenciales hardcoded visibles en el repositorio
- JWT_SECRET: `zXw9NYP8Qlg2flJSXn9O2oSXyqhyRgK6ok2oQ46SazRoQMAX5awNxx0iuwCBkQGG`
- DB_PASSWORD: `10TESORO+`
- APP_KEY: `base64:2sLLb9Z6ZOFpZNprnDAmfCNXihIREDK3oO+/B+0H6BU=`

### 2. CONFIGURACIÓN CORS PERMISSIVA
**Severidad: ALTA**
- `config/cors.php` línea 23: `allowed_origins_patterns` => `["#.*#"]`
- Esto permite CORS desde CUALQUIER origen
- `supports_credentials: true` combinado con `allowed_origins_patterns` muy permisivo

### 3. INYECCIÓN DE COMANDOS EN BACKUP RESTORE
**Severidad: CRÍTICA**
- `BackupController.php` líneas 160-168: Construcción de comando shell
- El path `$mysqlPath` viene de config sin validación adicional
- Se ejecuta proceso MySQL con privilegios directos

### 4. FILE UPLOAD SIN VALIDACIÓN MIME
**Severidad: ALTA**
- `PendriveController.php` línea 26: Solo valida max size pero NO tipo de archivo
- Permite subir cualquier archivo incluyendo ejecutables
- El nombre original se preserva sin sanitización

### 5. CONFIGURACIÓN DE SESIÓN
**Severidad: MEDIA**
- `config/session.php`:
  - `same_site: lax` (debería ser strict)
  - `SESSION_SECURE_COOKIE` no está forzado
  - `SESSION_DOMAIN` no definido

### 6. EXCLUSIONES CSRF
**Severidad: MEDIA**
- `VerifyCsrfToken.php` excluye rutas sin justificación clara

## Acciones Recomendadas
1. Rotar secrets en .env inmediatamente
2. Configurar CORS restrictivo
3. Agregar validación MIME en uploads
4. Revisar construcción de comandos MySQL
